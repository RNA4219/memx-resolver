# Security Policy

## 脆弱性報告窓口

- 連絡先: `security@memx.local`
- 件名: `[memx][security] <概要>`
- 記載必須: 再現手順、影響範囲、想定される悪用シナリオ、修正案（任意）
- 公開 Issue への投稿は避け、まずは上記窓口へ連絡してください。

## 初動 SLA

- 受領通知: **24時間以内**
- 一次トリアージ（重大度判定）: **3営業日以内**
- 修正方針の共有（回避策含む）: **7営業日以内**
- 高危険度（Critical/High）暫定対処: **14日以内** を目標

## 公開ポリシー

- 修正公開前は、脆弱性の詳細（PoC/再現コード/具体的悪用手順）を非公開で扱います。
- 修正後に、必要に応じて `CHANGES.md` と関連ドキュメントで概要を公開します。
- 公開時は以下を最低限含めます。
  - 影響バージョン
  - 重大度
  - 修正内容の要約
  - 利用者が取るべきアクション
