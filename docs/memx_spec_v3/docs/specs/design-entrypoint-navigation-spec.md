# design entrypoint navigation spec

利用者ペルソナごとの参照導線と、各文書冒頭に配置するナビゲーションブロックの統一仕様を定義する。

## 1. 利用者ペルソナごとの参照順（必須）

### 1.1 CLI 利用者

以下の順序を固定し、途中の分岐・省略を禁止する。

1. `spec.md`
2. `requirements.md`
3. `interfaces.md`（CLI節）
4. `contracts/cli-json.schema.json`

### 1.2 API 利用者

以下の順序を固定し、途中の分岐・省略を禁止する。

1. `spec.md`
2. `requirements.md`
3. `interfaces.md`（API節）
4. `contracts/openapi.yaml`

## 2. 各文書冒頭への必須ナビゲーションブロック定義

### 2.1 見出し名（固定）

- 見出し名は `## Navigation` に固定する。

### 2.2 リンク粒度（固定）

- リンクは「文書単位 + 節単位」の 2 段を上限とする。
- `interfaces.md` へのリンクは必ず対象節（CLI節またはAPI節）を含める。
- `contracts` 配下はファイル単位リンクを必須とする。

### 2.3 最大行数（固定）

- ナビゲーションブロックは 12 行以内とする（見出し行含む）。
- 12 行を超える場合は説明文を削除し、導線リンクのみを残す。

### 2.4 禁止事項（固定）

- 重複記述の禁止: 本文に同一導線を再掲しない。
- 正本逸脱の禁止: 導線・用語・順序は本仕様 1 章を正本とし、別順序や別名を許容しない。

## 3. 追従更新ルール

- `spec.md` 更新時は、同一変更セット内で `README.md` と各文書冒頭の `Navigation` ブロック整合を必須チェックとする。
- 整合チェックでは、少なくとも以下を確認する。
  - 導線順が本仕様 1 章と一致すること。
  - 参照ファイル名・節名が最新であること。
  - `README.md` の入口案内が導線ブロックと矛盾しないこと。

## 4. 検証方法

レビュー時は以下 3 観点を必須確認項目とする。

1. リンク到達性
   - ナビゲーションブロック内の全リンクが対象ファイル・対象節へ到達できること。
2. 正本一致
   - 導線順、見出し名、リンク粒度、最大行数、禁止事項が本仕様と一致すること。
3. 重複記述有無
   - 同一導線の本文再掲や、導線と競合する別導線が存在しないこと。
