# 品質ゲート定義

本書は、memx リポジトリで運用対象とする言語ごとの品質ゲート（lint / type / test）と、適用判定ルールを定義する。

## 対象言語ごとの適用有無

| 言語 | 運用対象 | 判定根拠 | 必須ゲート |
| --- | --- | --- | --- |
| Go | 対象 | `memx_spec_v3/go/go.mod` が存在する | `go test ./...` |
| Python | 対象外 | リポジトリ直下に `pyproject.toml` / `pytest.ini` が存在しない | なし（対象外） |
| Node.js | 対象外 | リポジトリ直下に `package.json` が存在しない | なし（対象外） |

## 適用判定ルール

1. 対象判定は「言語ランタイムの存在」ではなく「当該言語の運用設定ファイルの存在」で行う。
2. 設定ファイルがない言語は、Task の `Commands` に lint/type/test を必須記載しない。
3. Go は現行運用対象のため、`Commands` には `go test ./...` を必須で含める。
